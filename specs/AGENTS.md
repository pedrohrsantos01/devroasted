# Specs

- Antes de implementar uma feature nova, crie `specs/<feature>-spec.md`.
- O spec serve para registrar contexto, decisao e plano de execucao; nao para implementar direto.
- Estrutura minima:
  - `# <Feature name>`
  - `## Objective`
  - `## Current project context` ou `## Fontes consideradas`
  - `## Requirements`
  - `## Recommendation`
  - `## Implementation shape` ou `## File impact`
  - `## Risks` e/ou decisoes em aberto
  - `## TODO`
- Sempre cite arquivos, telas, libs e restricoes reais do projeto.
- Prefira bullets curtos, decisoes explicitas e TODOs acionaveis.
